Search Result Page Test - Effort Parity

Instructions

Overview

Help us determine how relevant results are to their queries. 

Steps

  • Read and understand the query, google for it if you are not sure
  • Consider the gender filter setting, assume that you visit a shopping site and chose this gender filter setting for your search
  • Review the search results of each query
  • Judge the level of relevancy for each result page
  • Answer the questions based on your judgement

Rules & Tips

Choose appropriate reasons for why for something is good or bad. 

Here are  explanations for the standard reasons:


  • Brand: only applicable, if the query specifies a brand (e.g. nike, adidas). If so, then brand is correct for some individual search result, if the result is from the specified brand, otherwise the brand is wrong.
  • Color: only applicable, if the query specifies a color. If so the color is right for some result, if the individual search result has this color, otherwise the color is wrong. 
  • Detail: only applicable, if the query specifies some detail like a color pattern or the style of some detail like the type of collar of a shirt or the heel of a shoe. The detail is correct for some individual search result, if the result exhibits this detail, otherwise detail is wrong.
  • Fit: only applicable, if the query specifies the fit of a product (e.g. slim fit, belly-free, plus size). The fit is correct for some individual search result, if the result exhibits this fit, otherwise the fit is wrong.
  • Gender: always applicable as the instructions provide a gender. The gender is correct for some individual search result, if the result is suitable for the given gender, otherwise the gender is wrong.
  • Material: only applicable, if the query specifies a material (e.g. jeans) or a property of a material (e.g. waterproof). The material is correct for some individual search result, if the result is made from such a material, otherwise the material is wrong.
  • Model: only applicable, if the query specifies a specific product model (e.g. adidas nmd r2) or range of models (e.g. nmds). If so then model is correct for some individual search result, if the result is this model / contained in the range of models, otherwise model is wrong.
  • Occasion: only applicable, if the query specifies an occasion (e.g. wedding, outdoor, swimming, raining) for which the product will be used. The occasion is correct for some individual search result, if the result is suitable for the occasion, otherwise the occasion is wrong.
  • Product Category: only applicable, if the query specifies a category of product (e.g. jacket, trousers). If so then product category is correct for some individual search result, if the result is of this product category, otherwise product is wrong.
  • Season: only applicable, if the query specifies a season. The season  is correct for some individual search result,  if the result is suited for this season, otherwise season is wrong.
  • Shape: only applicable, if the query specifies a shape (e.g. one shoulder dress, long dress). The shape is correct for some individual search result, if the result exhibits the shape, otherwise the shape is wrong.
  • Size: only applicable, if the query specifies a size for the product (e.g. shoes size, dress size). The size is correct for some individual search result, if the result has this size, otherwise the size is wrong.
  • Special Assortment: only applicable, if the query specifies a special assortment (e.g. maternity). The special assortment is correct for some individual search result, if the result belongs to this assortment, otherwise the special assortment is wrong.
  • Style: only applicable, if the query specifies a style of clothing (e.g. Boho, Hipster, 50s). The style is correct for some individual search result, if the result represents this style, otherwise the style is wrong.
